The Drawer Component

This is a collapsible panel that can be used for navigation, user preferences or other controls. On wider screens, it is fixed to the left of the browser window, and on small screens it's fixed to the top. (In IE8 and below, due to the lack of support for media queries, the drawer is always fixed to the top of the viewport)

It is opened by a click event on the drawer, and closed with a click event anywhere outside of it.

The show/hide events are attached with the $(".drawer").jnjDrawer() jQuery method.

JNJUI

JNJUI is a library of user interface patterns and components for use on Johnson and Johnson internal web applications.

Usage

JNJUI is intended to be included as-is, and extended or modified in your project's CSS or JavaScript. This will ensure that future updates and bugfixes to the project can be integrated as smoothly as possible. If you'd like to browse the unminified source, check out the src directory, but for production, use the packaged files in dist.

Required: Include the JNJUI CSS file in your document.

<link rel="stylesheet" href="dist/assets/stylesheets/jnjui.css" />

Optional: If you're using the sidebar drawer, accordion or modal components, include the relevant JavaScript files.

<script src="dist/vendor/javascripts/jquery.js"></script>
<script src="dist/vendor/javascripts/qbox.js"></script>
<script src="dist/assets/javascripts/jnj-accordion.js"></script>
<script src="dist/assets/javascripts/jnj-drawer.js"></script>

For support in IE8 or below, include the HTML5shiv in conditional comments.

<!--[if lte ie 8]>
<script src="dist/vendor/javascripts/html5shiv.js"></script>
<![endif]-->

Browser Support

Full support in the latest versions of:

  • Chrome

Basic support is included for Internet Explorer 8, but there aren't any extraneous JavaScript polyfills included to patch support for features like media queries, form placeholders, box shadows or the like.

The .mod Layout Component

A reusable, nestable layout pattern for creating distinct content sections in your application. For example, each section of this documentation is a .mod. The most basic implementation of a .mod is composed of a block-level parent element with the class mod. You can include optional modifier classes, as well as the optional child elements .mod_header and .mod_body.

Example Markup

<section class="mod">
    <header class="mod_header">
        <h2>Title</h2>
    </header>
    <div class="mod_body">
        <p>Pellentesque habitant morbi tristique senectus et netus et malesuada fames ac turpis egestas.</p>
    </div>
</section>

Highlight

This is a nested .mod with the m_highlight modifier class on the parent element. It adds the background color.

Pop

This is a nested .mod with the m_pop modifier class on the parent element. It adds the box-shadow.

Highlight & Pop

These modifier classes extend each other and can be combined.

Modifier classes may also be created to style different types of content. For instance, adding the m_list class to the parent elelment changes padding in the .mod_body element for nicer list formatting. Feel free to create new modifier classes following the pattern .m_[context].

Accordion List

Accordion lists are typically used inside of an m_list component. Their structure consists of a parent .acdn element that contains a single .acdn_header and a single .acdn_body element. The content of the acdn_body elements do not necessarily need to be list items.

Initilaize accordion components in your project's JavaScript file with $(".acdn").jnjAccordion().

Modals

You can control modals with the same classes used for other grid and .mod components.

The sample below is invoked with the following syntax:

$("#launch_sample_modal").on("click", function(){
    var qBox = new QBox({
        html: $("#sample_modal").html(),
        className: "mod m_pop m_modal l_col_6"
    });
    qBox.show();
});

This is the default options object. All can be overridden.

{ 
    html : "",
    modal : false,
    showClose : true,
    onOpen : function(){},
    onClose : function(){},
    className : "",
    closeHTML : "×",
    maskId : "qbMask",
    contentId : "qbContent",
    closeId : "qbClose",
    autoCenter : true,
    blurBackground : false
}
